Meridian Components

Current-State Architecture Review

Engagement: RFP Modernization (R4)

Executive Summary

This document details the system architecture left behind by the previous vendor. We have conducted a complete code audit to map out component boundaries, data-flows, and integration points. This architectural baseline represents a lightweight, functional prototype stack suitable for demo purposes, but has clear bottlenecks and structural patterns that we must remediate to prepare it for production.

System Topology Map

Client Layer (Vue 3)

Runs in-browser. Built with Vite and powered by Vue 3's Composition API. Axios centralizes API requests inside api.js. Core states (Auth, Language, Dashboard Filters) are managed cleanly using standalone reactive Composables.

Service Layer (FastAPI)

High-performance API built with FastAPI. Features automatic OpenAPI (/docs) generation. Input payloads and return records are strictly typed and validated at the boundary using Pydantic models. Handles routing, query filters, and data manipulation.

Data Layer (Flat Files)

The database consists of flat JSON files stored inside server/data/. During startup, mock_data.py loads these tables into memory. Changes live strictly in memory and are discarded when the FastAPI process is restarted.

Architecture Strengths

  • ✓ Decoupled Structure: Complete separation between the UI application and backend. Ready-made for deployment across CDN systems.
  • ✓ Strong API Contract: Pydantic validation on the backend ensures robust typing. If the API contract is violated, the FastAPI boundary returns standard errors rather than letting bugs bleed into the logic.
  • ✓ Elegant Frontend Utility: Utilizing Vue Composables (like useFilters.js) cleanly isolates logic from template markups.

Technical Debt & Security Gaps

  • ✗ State Persistence Vulnerability: The system's "database" is strictly in-memory. Writes do not persist back to disk. This is a blocker for any real-world pilot.
  • ✗ No Authentication/RBAC: The API is completely unprotected. Any caller can hit any endpoint. There is no user identity propagation or token enforcement in place on the backend.
  • ✗ Dead API Codes: The Vue client (api.js) references dead endpoints on the backend like /api/tasks. These do not exist in server/main.py, causing network errors during execution of certain sub-components.

Modernization Roadmap

Phase 1: Immediate Remediation (Active Engagement)

Fix defective screens (like the Reports View which currently fails to load or graph trends correctly). Clean up inconsistent filter parameters across the API client, and align backend routes to resolve dead endpoint references (such as the task management features).

Phase 2: Database Migration (Target State)

Swap the mock_data.py in-memory JSON file loader for a lightweight transactional SQL database (e.g. SQLite for lightweight operations, or PostgreSQL for scaling). Introduce SQLAlchemy / SQLModel ORM to manage database transactions safely.

Phase 3: Identity & Security Integration

Secure the REST API. Introduce JWT-based OAuth2 flows inside FastAPI, write custom security dependencies to validate caller tokens, and establish Role-Based Access Control (RBAC) to differentiate Warehouse Operators from procurement executives.
